Co-op Card Auto-Battler

NOTE! Hobby project / Learning project In this project I wanted to try out Unitys Netcode for GameObjects (NGO) multiplayer framework, so I built a simple co-op card auto-battler prototype. The hosting and joining is handled completely by Unitys Lobby and Relay services.

Development GIFs

This demo is captured from inside Unity with their Multiplayer play mode tools. The left editor acts as host and the right editor is the client that joins the host's lobby. The host chooses from a range of different commander cards that have different abilities and synergies.

Here the client player joins the host. The host copies their lobby code (top left corner of left editor), once the client inputs the code they are able to connect and play with the host.

Here I'm showcasing a pretty cool holographic effect that I made using Unity's shader graph. When the player mouses over a card the game showcases that card closer and rotates the card and this makes the holographic effect look nice. (The cards are 3d objects in the game!)

The players occupy their side of the board with units they can buy from the shop. Aquiring three units of the same type will merge them into a stronger version. (Three tiers: Bronze, Silver, Gold)

Here I'm showing off the networking part of the game the movement of the cards is replicated across the network. So when one player is moving their card around both players see the card moving in the same way. (The host is moving a card here)

Showcase of the combat in the game. The units on the board will automatically attack each other when combat is initiated. Attack order is determined by the units' speed stat. And units will attack the closest enemy unit by default. Units will also get buffs if there are many units of the same type on the board in this example there are 5 unique humans on the board.

When combat is done the players can proceed together a new encounter loads up and the board state is shared between the players. Players are also free to reposition their units on the board before starting combat.

Final Thoughts

This was a fun project to work on and it was a great way to learn the basics of Unity's multiplayer framework. Building replicated objects and systems require an additional layer of thinking to make sure the game state is consistent across the network and that the game is responsive for all players. Specifically thinking about what should be server authorative and what should be client authorative is an important part of building a multiplayer game. And with this project i got to practice that type of thinking!
